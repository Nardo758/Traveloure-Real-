================================================================================
API ENDPOINTS
================================================================================

1. Search Venues
   GET /api/venues/search?location={city}&type={type}
   
   Parameters:
   - location (required): City name
   - type (optional): wedding_venue, hotel, restaurant, venue
   - keyword (optional): Custom search term
   - minRating (optional): 0, 3.5, 4.0, 4.5
   - radius (optional): Search radius in meters

2. Get Venue Details
   GET /api/venues/:placeId
   
   Returns full venue details including opening hours

3. Search Wedding Vendors
   GET /api/venues/wedding-vendors?location={city}&vendorType={type}
   
   Vendor types: photographer, florist, caterer, dj, planner,
                 videographer, makeup, baker

================================================================================
TROUBLESHOOTING
================================================================================

Issue: No venues showing up
Fix:
  1. Check .env has GOOGLE_MAPS_API_KEY
  2. Verify API key at: https://console.cloud.google.com/apis/credentials
  3. Enable: Places API, Maps JavaScript API, Geocoding API
  4. Restart server: npm run dev

Issue: "Failed to fetch venues" error
Fix:
  1. Try different location (e.g., "New York" vs "NYC")
  2. Check Google Cloud Console quotas
  3. Test API directly: /api/venues/search?location=New York&type=venue
  4. Check browser Network tab for errors

Issue: Slow results
Fix:
  - First search after cache expiry is slower (1-3 seconds)
  - Second search is instant (cached)
  - This is normal and expected

Issue: Photos not loading
Fix:
  - Some venues don't have photos
  - Component handles this gracefully with fallback image
  - No action needed
