## 1. TWO VIEWS, SAME DATA

### Itinerary Tab — Transport in context
The user reads their trip day-by-day. Between every two activities, a transport
leg connector shows the selected mode, duration, and cost. The user picks their
mode here because they're thinking about their day flow.

### Transport Tab — Transport operations center
A flat dashboard of ALL transport legs across the entire trip. The user manages
bookings here: "I have 12 legs, 3 booked, 9 still need arranging. Should I get
a metro pass?" Includes booking options (platform + affiliate) and multi-day
pass recommendations.

Both views read from and write to the same `transport_legs` table. A mode
change in one view is instantly reflected in the other.

### Component: InlineTransportSelector

**File:** `client/src/components/itinerary/InlineTransportSelector.tsx`

**Props:**
```typescript
interface InlineTransportSelectorProps {
  leg: {
    id: number;
    fromName: string;
    toName: string;
    distanceDisplay: string;
    distanceMeters: number;
    recommendedMode: string;
    userSelectedMode: string | null;
    estimatedDurationMinutes: number;
    estimatedCostUsd: number | null;
    alternativeModes: {
      mode: string;
      durationMinutes: number;
      costUsd: number | null;
      energyCost: number;
      localName?: string;
    }[];
    googleMapsUrl: string;
    appleMapsUrl: string;
  };
  readOnly?: boolean;     // public share view = read only
  onModeChange?: (legId: number, mode: string) => void;
}
```

**Behavior:**

1. **Displays all available modes as radio options** — not chips, not a dropdown. 
   Radio buttons in a compact vertical list so the user can see all options at 
   once with duration and cost.

2. **Pre-selects the current mode:**
   - If `userSelectedMode` is set → that's selected
   - Otherwise → `recommendedMode` is selected
   - The recommended mode always shows a "✓ Best" or "Recommended" tag

3. **On mode change:**
   - Optimistic UI: updates immediately
   - Calls `PATCH /api/transport-legs/:legId/mode` with `{ selectedMode: "taxi" }`
   - On success: keeps the new mode, shows brief toast if timing shifted
     ("Taxi saves 6 min — next activity can start earlier")
   - On failure: reverts to previous mode, shows error toast

4. **Mode list comes from the leg data:**
   - The recommended mode is always first with its duration/cost
   - `alternativeModes` array provides the rest
   - Each mode shows: icon + local name + duration + cost
   - Walking only appears if distance < 3 km

5. **"Open in Maps" link** at the bottom uses the SELECTED mode
   (not the recommended mode). When user switches to taxi, the maps
   link changes to driving directions.

6. **Read-only mode:** Public share viewers see the selected mode
   highlighted but cannot change it. Radio buttons become static display.

7. **Collapsed state (optional UX enhancement):**
   Default: show just the selected mode in a single line:
   `🚃 Train • 18 min • $2.50`
   Click to expand and see all options.
   This keeps the itinerary scannable without transport dominating the view.

| Aspect | Itinerary Tab | Transport Tab |

### Booking option behavior in Transport tab

**Platform options ("Book on Traveloure"):**
- Green badge, same as itinerary tab activities
- [Book — $X] button → initiates booking flow
- Creates booking record → Stripe checkout → confirmation
- After booking: status changes to green "Confirmed" badge
- Commission: 4-12% to Traveloure based on provider tier

**Partner options (any affiliate):**
- Blue badge with partner name (↗ Viator, ↗ Klook, ↗ 12Go, ↗ GetYourGuide)
- Which partners appear depends on:
  - What MODE the user selected for this leg
  - What's available for this DESTINATION and ROUTE
  - What affiliate integrations are active
- [View on X] button → logs click for affiliate tracking → opens partner site in new tab
- After user returns: "Mark as booked" option with confirmation ref input
- Commission: 8-15% affiliate revenue

**Rideshare deep links (Uber, Grab, Bolt, Ola):**
- Blue badge with "↗ Open App"
- Only appears when selected mode is taxi/rideshare
- Pre-fills pickup and dropoff coordinates
- Mobile: opens native app. Desktop: opens web version.
- No "mark as booked" — rideshare is on-demand

**Walking / Free options:**
- "No booking needed" message
- Just shows the maps link for walking directions

## 4. DATA MODEL

### New table: `transport_booking_options`

```typescript
export const transportBookingOptions = pgTable("transport_booking_options", {
  id: serial("id").primaryKey(),

  // Which transport leg this option applies to
  transportLegId: integer("transport_leg_id")
    .references(() => transportLegs.id, { onDelete: "cascade" }),

  // OR: multi-day pass linked to trip, not a specific leg
  tripId: integer("trip_id")
    .references(() => trips.id, { onDelete: "cascade" }),

  // What kind of booking
  bookingType: text("booking_type").notNull(),
  // "platform"   → Book on Traveloure (Service Provider)
  // "affiliate"  → Partner site (Viator, Klook, 12Go, GetYourGuide, etc.)
  // "deep_link"  → Rideshare app (Uber, Grab, Bolt, Ola)
  // "info_only"  → Walking, self-arranged (no booking needed)

  // Which partner/provider
  source: text("source").notNull(),
  // "traveloure", "viator", "getyourguide", "klook", "12go",
  // "uber", "grab", "bolt", "ola", "walking", "self_arranged"

  // Which transport mode this option covers
  modeType: text("mode_type").notNull(),
  // Must match one of the modes from transport profiles
  // Options only show when the user has selected THIS mode for this leg

  // Display
  title: text("title").notNull(),
  description: text("description"),
  iconType: text("icon_type"),

  // Pricing
  priceDisplay: text("price_display"),         // "$75" or "$35-55" or "Free"
  priceCentsLow: integer("price_cents_low"),
  priceCentsHigh: integer("price_cents_high"),
  pricePerPerson: boolean("price_per_person").default(false),
  currency: text("currency").default("USD"),

  // Timing
  estimatedMinutes: integer("estimated_minutes"),

  // Rating (platform providers + affiliates with reviews)
  rating: doublePrecision("rating"),
  reviewCount: integer("review_count"),

  // Platform provider link
  providerId: integer("provider_id")
    .references(() => providers.id),

  // External links
  externalUrl: text("external_url"),
  affiliateCode: text("affiliate_code"),
  deepLinkScheme: text("deep_link_scheme"),

  // Status
  bookingStatus: text("booking_status").default("available"),
  // "available" | "booked" | "confirmed" | "cancelled"
  bookingId: integer("booking_id")
    .references(() => bookings.id),
  externalConfirmationRef: text("external_confirmation_ref"),

  // Multi-day pass fields
  isMultiDayPass: boolean("is_multi_day_pass").default(false),
  passValidDays: integer("pass_valid_days"),
  savingsVsIndividualCents: integer("savings_vs_individual_cents"),

  // Sorting + recommendation
  sortOrder: integer("sort_order").default(0),
  isRecommended: boolean("is_recommended").default(false),

  createdAt: timestamp("created_at").defaultNow(),
  updatedAt: timestamp("updated_at").defaultNow(),
});
```

### Critical relationship: mode filtering

Booking options are stored for ALL possible modes per leg. But the Transport
tab only DISPLAYS options matching the mode the user selected in the Itinerary
tab.

Example for a Kyoto leg:
```
transport_leg (id: 101)
  ├── userSelectedMode: "train"
  │
  ├── booking_option (modeType: "train", source: "traveloure", title: "JR Day Pass")
  ├── booking_option (modeType: "train", source: "klook", title: "JR Kansai Pass")
  ├── booking_option (modeType: "train", source: "12go", title: "Single Tickets")
  ├── booking_option (modeType: "taxi", source: "uber", title: "Uber Japan")
  ├── booking_option (modeType: "taxi", source: "traveloure", title: "Private Transfer")
  └── booking_option (modeType: "bike", source: "klook", title: "Bike Rental")
```

When user selects "train" → Transport tab shows: JR Day Pass, JR Kansai Pass, Single Tickets
When user switches to "taxi" → Transport tab shows: Uber Japan, Private Transfer
When user switches to "bike" → Transport tab shows: Bike Rental

## 5. API ENDPOINTS

### 5.1 Get Transport Hub Data

```
GET /api/itinerary/:tripId/transport-hub
```

Returns summary + all legs grouped by day + multi-day passes.

Server-side: for each leg, filters `transport_booking_options` by the leg's
current selected mode (`userSelectedMode || recommendedMode`).

### 5.2 Book Platform Option

```
POST /api/transport-booking-options/:optionId/book
```

Body: `{ travelers: 2, date: "2026-04-01", specialRequests: "..." }`

Creates booking → initiates Stripe checkout → returns checkout URL.

### 5.3 Track Affiliate Click

```
POST /api/transport-booking-options/:optionId/click
```

Logs click for affiliate commission tracking.
Returns `{ redirectUrl: "https://klook.com/...?affiliate=traveloure" }`

### 5.4 Mark Third-Party as Booked

```
PATCH /api/transport-booking-options/:optionId/status
```

Body: `{ bookingStatus: "booked", confirmationRef: "KLK-2026-ABC" }`

User confirms they completed the external booking.

### 5.5 Mode Change (existing — already built)

```
PATCH /api/transport-legs/:legId/mode
```

Body: `{ selectedMode: "taxi" }`

When the mode changes:
1. Updates `userSelectedMode` on the leg
2. The Transport tab auto-refreshes and shows booking options for the new mode
3. Maps URLs regenerate for the new mode

## 6. POPULATING BOOKING OPTIONS

### When it runs

After the Transport Leg Calculator computes legs (post-optimization),
a second pass populates booking options for ALL modes on each leg:

```typescript
async function populateBookingOptions(
  tripId: number,
  legs: TransportLeg[],
  destination: string,
  travelers: number
): Promise<void> {

  for (const leg of legs) {
    const allOptions: InsertTransportBookingOption[] = [];

    // For EACH available mode on this leg, find booking options
    const allModes = [
      { mode: leg.recommendedMode, ...getRecommendedDetails(leg) },
      ...leg.alternativeModes
    ];

    for (const modeOption of allModes) {

      // 1. PLATFORM — find Traveloure Service Providers for this mode
      const providers = await findTransportProviders(
        destination, modeOption.mode,
        leg.fromLat, leg.fromLng, leg.toLat, leg.toLng
      );
      for (const provider of providers) {
        allOptions.push({
          transportLegId: leg.id,
          bookingType: "platform",
          source: "traveloure",
          modeType: modeOption.mode,
          title: provider.businessName,
          description: provider.serviceDescription,
          priceDisplay: formatPrice(provider.calculatePrice(leg.distanceMeters)),
          priceCentsLow: provider.calculatePrice(leg.distanceMeters) * 100,
          estimatedMinutes: modeOption.durationMinutes,
          rating: provider.ratingAvg,
          reviewCount: provider.reviewCount,
          providerId: provider.id,
          sortOrder: 0,
          isRecommended: modeOption.mode === leg.recommendedMode,
        });
      }

      // 2. AFFILIATES — find partner options for this mode + destination
      const affiliates = await findAffiliateOptions(
        destination, modeOption.mode,
        leg.fromName, leg.toName,
        leg.distanceMeters
      );
      for (const aff of affiliates) {
        allOptions.push({
          transportLegId: leg.id,
          bookingType: "affiliate",
          source: aff.partner,
          modeType: modeOption.mode,
          title: aff.title,
          description: aff.description,
          priceDisplay: aff.priceDisplay,
          priceCentsLow: aff.priceCentsLow,
          pricePerPerson: aff.perPerson,
          estimatedMinutes: modeOption.durationMinutes,
          externalUrl: aff.buildAffiliateUrl(),
          affiliateCode: aff.trackingCode,
          rating: aff.rating,
          reviewCount: aff.reviewCount,
          sortOrder: 1,
        });
      }

      // 3. RIDESHARE DEEP LINKS — only for taxi/rideshare modes
      if (["taxi", "rideshare"].includes(modeOption.mode)) {
        const apps = getRideshareApps(destination);
        for (const app of apps) {
          allOptions.push({
            transportLegId: leg.id,
            bookingType: "deep_link",
            source: app.name,
            modeType: modeOption.mode,
            title: app.displayName,
            description: `Est. ${modeOption.durationMinutes} min`,
            priceDisplay: app.estimateRange(leg.distanceMeters),
            priceCentsLow: app.estimateLow(leg.distanceMeters) * 100,
            priceCentsHigh: app.estimateHigh(leg.distanceMeters) * 100,
            estimatedMinutes: modeOption.durationMinutes,
            deepLinkScheme: app.buildDeepLink(
              leg.fromLat, leg.fromLng, leg.toLat, leg.toLng
            ),
            sortOrder: 2,
          });
        }
      }

      // 4. WALKING — info only, no booking
      if (modeOption.mode === "walk") {
        allOptions.push({
          transportLegId: leg.id,
          bookingType: "info_only",
          source: "walking",
          modeType: "walk",
          title: "Walk",
          description: `${leg.distanceDisplay} • Scenic route`,
          priceDisplay: "Free",
          priceCentsLow: 0,
          estimatedMinutes: modeOption.durationMinutes,
          sortOrder: 3,
        });
      }
    }

    // Bulk insert all options for this leg
    if (allOptions.length > 0) {
      await db.insert(transportBookingOptions).values(allOptions);
    }
  }

  // 5. MULTI-DAY PASSES
  await populateMultiDayPasses(tripId, destination, legs, travelers);
}
```
